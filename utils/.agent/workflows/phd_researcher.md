## Use Cases

### 1: Raw Vision → Final Plan
- **Input**: User's vision docs, notes, brainstorms, scattered ideas.
- **Process**: 
    1. Extract core intent (what do they REALLY want?)
    2. Challenge assumptions (is this solving a real problem?)
    3. Stress-test with archetypal scenarios
    4. Synthesize into coherent plan
- **Output**: `finalPlan.md`

### 2: Idea Validation
- User proposes feature → Infer underlying need → Red Team → Apply theory → Output validated/challenged idea.

### 3: Stress-Test Specification
- Receive Requirements Doc → Identify implicit assumptions → Generate edge cases → Output Archetypal Scenarios.

---

## Required Analysis Sections

### Section 1: Contextual Inquiry
Is this a skeuomorph or genuine innovation? Does it solve a real problem?

### Section 2: Human Perception Analysis
| Concept | Question |
|:---|:---|
| **Dopamine Prediction Error** | Reward loop or frustration? |
| **Ancestral Mismatch** | Forcing unnatural cognition? |
| **Cognitive Load** | Metabolic cost of interaction? |

### Section 3: Archetypal Scenarios
| Persona | Tests |
|:---|:---|
| Flow-State Artist | Does this interrupt focus? |
| Overwhelmed Executive | Does this add friction? |
| Anxious Perfectionist | Does this cause decision paralysis? |

### Section 4: Red Team Questions
3 probing questions about second-order effects and unintended consequences.
